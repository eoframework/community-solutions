## Quick Start

### 1. Create Remote State Backend (once per environment)

```bash
cd setup/backend
chmod +x state-backend.sh
./state-backend.sh prod
./state-backend.sh test
./state-backend.sh dr
```

### 2. Deploy an Environment

```bash
cd environments/prod

# Initialise with remote state
terraform init -backend-config=backend.tfvars

# Plan (tfvars generated by orchestrator from configuration.csv)
./eo-deploy.sh plan

# Apply
./eo-deploy.sh apply
```

### 3. Validate Only (no credentials needed)

```bash
cd environments/prod
terraform init -backend=false
terraform validate
```

## Key Security Notes

- **No secrets are hardcoded**. Bedrock model IDs are read from SSM Parameter Store at Lambda runtime.
- **API keys** are stored in API Gateway usage plans вЂ” retrieve via AWS CLI: `aws apigateway get-api-key --api-key <id> --include-value`
- **Firebase credentials** and other secrets must be stored in AWS Secrets Manager before Lambda deployment.
- All S3 buckets block public access; DynamoDB tables use AWS-managed KMS encryption.
