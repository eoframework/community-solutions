# Amatra Agentic Orchestration Platform — Terraform Automation

Infrastructure-as-Code for the Amatra Agentic Orchestration Platform on AWS.

## Architecture

```
terraform/
├── environments/
│   ├── prod/          # Production environment
│   ├── test/          # Test / development environment
│   └── dr/            # Disaster Recovery environment
└── modules/
    ├── aws/           # Tier 1: AWS provider primitives
    │   ├── cognito/
    │   ├── api-gateway/
    │   ├── lambda/
    │   ├── dynamodb/
    │   ├── s3/
    │   ├── waf/
    │   └── ecr/
    ├── security/      # Tier 2: WAF + CloudTrail + Secrets Manager
    ├── storage/       # Tier 2: S3 buckets + ECR
    ├── database/      # Tier 2: DynamoDB tables
    ├── api/           # Tier 2: Lambda handlers + API Gateway
    ├── cognito/       # Tier 2: Cognito User Pool + post-confirmation Lambda
    ├── monitoring/    # Tier 2: CloudWatch dashboards + SNS
    └── cicd/          # Tier 2: CodeBuild projects
```

## Environments

| Environment | Purpose | Region |
|-------------|---------|--------|
| prod | Production platform serving all consultants | us-west-2 |
| test | Development / integration testing | us-west-2 |
| dr | Disaster Recovery (mirrors prod sizing) | us-west-2 |

## Quick Start

### 1. Set up remote state backend

```bash
cd scripts/
chmod +x state-backend.sh
./state-backend.sh prod
./state-backend.sh test
./state-backend.sh dr
```

### 2. Deploy an environment

```bash
cd environments/prod/
chmod +x ../../scripts/eo-deploy.sh

# Initialise with remote backend
terraform init -backend-config=backend.tfvars

# Plan (config values injected via orchestrator-generated tfvars in config/)
terraform plan -var-file=config/project.tfvars \
               -var-file=config/application.tfvars \
               -var-file=config/compute.tfvars \
               -var-file=config/database.tfvars \
               -var-file=config/security.tfvars \
               -var-file=config/networking.tfvars \
               -var-file=config/storage.tfvars \
               -var-file=config/quota.tfvars \
               -var-file=config/integration.tfvars \
               -var-file=config/monitoring.tfvars \
               -var-file=config/operations.tfvars

# Apply
terraform apply ...
```

### 3. Validate all environments

```bash
chmod +x scripts/validate-all.sh
./scripts/validate-all.sh
```

## Security

- Secrets (GitHub PAT, Cognito client secret) are stored in AWS Secrets Manager — **never** in tfvars files or Lambda environment variables
- Lambda execution roles follow least-privilege IAM — no wildcard resource ARNs
- All DynamoDB tables and S3 buckets are encrypted at rest (SSE-AES256)
- WAF WebACL with IP rate limiting and AWS Managed Rules protects the API Gateway stage
- CloudTrail captures all management and data-plane API calls

## No .tfvars Files

> **Note:** `.tfvars` files are generated by the orchestrator from `configuration.csv`. Do **not** commit or create `.tfvars` files manually.
